## 一、项目构思与技术调研

### 1.1 技术选型与架构设计
独立完成前端技术栈的选型评估和架构设计，确定以 Vue 3 + TypeScript + Vite + Pinia + Element Plus 为核心的前端方案，并完成以下工作：

- 评估对比 React / Vue 3 / Angular 等前端框架，结合团队技术背景和课程项目周期最终选定 Vue 3
- 设计前端分层架构（视图层 → 组件层 → 状态管理层 → API 封装层 → 工具层），确保各层职责清晰、耦合度低
- 设计前后端接口契约规范（RESTful / JWT / 统一响应结构 / 分页约定 / ISO 8601 时间格式）
- 设计 Mock 服务架构，使前后端可并行开发

### 1.2 开发环境与工程化基础设施搭建
- 初始化 Vue 3 + TypeScript + Vite 项目脚手架，配置 tsconfig、package.json 和 Vite 构建配置
- 配置 GitHub Actions CI 工作流（`.github/workflows/ci.yml`），实现 push 和 PR 自动触发的构建、单元测试和端到端测试
- 配置 Playwright 端到端测试框架（`playwright.config.ts`），集成 Chromium 浏览器自动化
- 设计并实现完整的 Mock 服务（`mock/`），覆盖认证、活动、日历、知识、健康检查等所有 API 模块

**涉及代码：**
- `.github/workflows/ci.yml`
- `playwright.config.ts`
- `vite.config.ts`
- `tsconfig.json`, `tsconfig.app.json`, `tsconfig.node.json`
- `mock/index.js`, `mock/db.js`, `mock/utils.js`
- `mock/routes/`（auth.js, activity.js, calendar.js, admin.js, health.js, knowledge.js, profile.js, upload.js）
