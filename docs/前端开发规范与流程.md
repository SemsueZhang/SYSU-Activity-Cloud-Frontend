# 前端开发规范与流程

## 目标

本项目的前端开发采用“先写设计文档，再进行网页开发”的工作方式。所有页面、布局调整和交互变更，在进入实现阶段之前，都应先完成对应的设计说明，再进入代码开发与联调。

## 原则

1. **模块边界清晰前不做大规模组件抽象。**  
   先在一个文件里写清楚，等出现明显重复再抽组件，不提前抽象。

2. **API 层只在实际使用时创建。**  
   不提前封装"未来可能用到"的接口。对应页面恢复时再建对应的 `api/*.ts`。

3. **Mock 与前端页面保持 1:1。**  
   恢复一个页面时只补该页面必需的 mock 端点，不一次性补全所有后端接口。

4. **路由只注册已实现的页面。**  
   未实现的入口在主页上给出明确提示（如"内容未实现！"），不保留跳转 404 的链接。

5. **不靠前端隐藏按钮做权限控制。**  
   前端只处理路由级的登录/未登录分流，角色级控制由后端校验。

6. **修改现有页面布局前，先确认不会破坏既有视觉设计。**  
   首页是当前唯一完整业务入口，改动首页前先确认影响范围。

7. **状态与待办按日期归档。**  
   `docs/frontend-plan/` 下的计划文件带日期前缀，旧文件不删除，反映当时认知。


## 基本原则

- 先设计，后开发：先明确页面结构、交互、数据来源和验收标准，再开始编写 `.vue` 页面或组件。
- 先文档，后代码：任何可见 UI 变化，都应先在设计文档中说明，再落到实现文件中。
- 设计与实现保持一致：代码实现必须以设计文档为依据，若实现过程中发现设计需要调整，应先更新设计文档。

## 标准流程

### 1. 需求确认

- 明确页面要解决的问题、目标用户和核心功能。
- 梳理页面入口、跳转关系、数据依赖和权限要求。
- 确认是否需要适配移动端、平板和桌面端。

### 2. 编写设计文档

设计文档建议放在 [docs/frontend-design](frontend-design) 目录下，文档内容至少包括：

- 页面目标与适用场景
- 布局结构与区域划分
- 交互行为与状态变化
- 数据来源与接口约定
- Mock 数据要求
- 响应式与可访问性要求
- 开发 TODO 列表

如果是单独页面，建议为每个页面建立独立设计文档，例如：

- [主页设计](frontend-design/HOMEPAGE_DESIGN.md)
- [登录页设计](frontend-design/LOGIN_DESIGN.md)
- [注册页设计](frontend-design/REGISTER_DESIGN.md)

### 3. 按设计开发页面

- 按设计文档中的布局和交互实现页面。
- 优先在单个页面文件中完成模板、脚本和样式。
- 如果需要抽取公共能力，优先放到 `frontend/src/components`、`frontend/src/stores` 或 `frontend/src/api`。
- 代码实现应严格对应设计文档中的模块划分与数据逻辑。

### 4. 联调与验证

- 使用本地 Mock 服务或后端接口完成联调。
- 验证页面状态、空数据、错误态、加载态和边界输入。
- 对照设计文档逐项检查 TODO 是否完成。

### 5. 文档同步与归档

- 如果实现与原设计有偏差，应回写设计文档。
- 页面完成后，将最终实现要点和注意事项补充到设计文档中。
- 新增页面或新交互时，先补设计文档，再开始开发。

## 页面开发清单

- [ ] 1. 明确页面目标和使用场景
- [ ] 2. 编写对应设计文档
- [ ] 3. 确认布局、交互、接口和 Mock 数据
- [ ] 4. 开始页面实现
- [ ] 5. 完成本地联调和错误检查
- [ ] 6. 补充测试说明或测试用例
- [ ] 7. 更新设计文档与项目文档索引

## 统一要求

- 新页面优先使用单文件组件实现。
- 设计文档应可直接指导开发，不应只写概念描述。
- 任何新增页面在开始编码前，都应先有对应设计文档。
- 若后续需要多人协作，设计文档应成为统一的实现依据。

## 推荐落地方式

- 页面设计文档统一放在 [docs/frontend-design](frontend-design) 下。
- 页面实现统一放在 `frontend/src/views` 下。
- 公共组件统一放在 `frontend/src/components` 下。
- 接口与数据层统一放在 `frontend/src/api` 下。
