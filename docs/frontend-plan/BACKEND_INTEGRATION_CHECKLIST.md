# 后端联调清单

本清单用于前后端联合测试。前端页面和路由始终保持 `Activity` 与 `/activities` 格式；任何后端资源名或字段差异均在网关/API 适配层处理。

## 启动前

- [ ] `VITE_API_BASE_URL` 指向后端或同源网关。
- [ ] CORS 允许 `Authorization`，验证码响应暴露 `X-Captcha-Token`。
- [ ] SPA history fallback 已配置，刷新任意前端路由不返回服务器 404。
- [ ] 明确时区为 `Asia/Singapore`，日期字段采用 ISO 8601。

## 冻结接口验证

- [ ] 验证码、登录、注册、发送邮箱验证码、`/auth/me`。
- [ ] 订阅、通知已读、日历事件、ICS 下载、关联信息、知识节点。
- [ ] 管理员审核、数据源、审计、导出与权限错误码。
- [ ] 分页返回包含 `items`、`total`、`page`、`per_page`；422 字段错误可定位或展示。

## 前端适配边界

- [ ] 将前端活动资源 `/activities` 适配为后端实际资源，保持页面与 TypeScript `Activity` 不变。
- [ ] 逐项确认“我的发布”的权限和查询语义，禁止前端据此推断越权数据。
- [ ] 对未冻结的报名、独立收藏、资料更新、忘记密码给出实际接口或明确暂不支持；未确认前不将 Mock 成功当作联调成功。
- [ ] 确认附件上传：`multipart/form-data`、文件类型/10 MB 限制、返回的 `id/name/url/mime_type/size`、删除权限和公开下载策略。
- [ ] 验证 401 清理登录状态并回跳，403 不泄露管理数据，404 和 422 有可读反馈。

## 完成条件

- [ ] 访客、viewer、publisher、admin 四类身份在真实后端下分别通过核心路径。
- [ ] 网络慢响应、空列表、401、403、404、422、500 不造成页面卡死或假成功。
- [ ] 将发现的字段、错误码和时区差异回写到 [契约矩阵](BACKEND_CONTRACT_MATRIX.md)。
