# 我的活动设计

## 路径与权限

- `/my/activities`：本人报名的活动，所有登录用户均可访问。
- `/my/created/approved`：本人创建且已通过的活动，仅 publisher/admin 可访问。
- `/my/created/reviewing`：本人创建且正在审核的活动，仅 publisher/admin 可访问。
- `/my/created/rejected`：本人创建且未通过或仍为草稿的活动，仅 publisher/admin 可访问。
- 管理员的全站审核仍位于独立管理页面。

## 布局与交互

- “我报名的活动”和“我创建的活动”不在同一页面；页面间通过明确导航按钮切换。
- “已通过”“在审核”“未通过”也分别使用独立 URL 和独立页面，每次只加载、展示当前状态。
- 各页面均按活动时间从新到旧排列；活动时间为空时置后。
- 已超过活动开始时间的项目显示灰色，其余项目显示绿色。
- 草稿显示在“未通过”页面，并保留独立的“草稿”标签。
- 草稿和已驳回活动可编辑、提交审核；已驳回活动醒目显示原因。
- “我报名的活动”支持取消报名，成功后同时从列表和个人日历中移除。

## 验收

- 普通登录用户可进入报名页面，但不能进入创建活动状态页面。
- publisher/admin 只看到本人创建的活动；各状态页面互不混排，排序、结束态颜色正确，操作后跳转到对应状态页面。
- 报名成功后活动同时出现在本页与日历，取消报名后两处同时消失。
