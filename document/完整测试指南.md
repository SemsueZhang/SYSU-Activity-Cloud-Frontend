# 校园活动信息平台完整测试指南

本文档用于本地验收前端、Mock、真实后端、数据库迁移与 GitHub CI。测试时 **Mock 和真实后端不能同时占用 5000 端口**。

## 一、环境与端口

| 服务 | 地址 | 用途 |
| --- | --- | --- |
| 前端 Vite | `http://localhost:3000` | 页面调试与联调 |
| Mock | `http://localhost:5000` | 前端快速回归 |
| Flask 后端 | `http://localhost:5000` | 真实接口联调 |

首次安装前端依赖：

```powershell
cd D:\ZhangChenrui\Projects\SYSU-Activity-Cloud-Frontend
npm install
```

## 二、Mock 回归测试

Mock E2E 不需要手动启动服务；Playwright 会自动启动 Mock（5000）和 Vite（3000）。确保 5000 未被真实后端占用后执行：

```powershell
cd D:\ZhangChenrui\Projects\SYSU-Activity-Cloud-Frontend
npm run build
npx vitest run --pool=threads --maxWorkers=1
npm run test:e2e
```

若浏览器组件缺失：

```powershell
npx playwright install chromium
```

Mock 自动回归应覆盖：搜索、注册登录、图形验证码、报名表、重复报名、取消报名、日程同步、草稿权限、发布审核、附件失败重试、CSV、ICS、关联活动和首页异常提示。

## 三、真实前后端联调

访问 `http://localhost:3000`。Vite 会将 `/api` 代理至真实后端 `5000`。

### 角色验收清单

1. 游客：公开活动、搜索、详情、关联活动正常；草稿、报名名单和导出 CSV 均返回 403/401 的权限错误。
2. 普通用户：注册、登录、忘记密码、收藏、订阅、填写姓名/学号/学院/邮箱报名；报名后自动出现在日历，取消报名后自动移除；可申请成为发起人。
3. 发起人：创建草稿、上传合规附件、编辑、提交审核、查看报名名单并导出 CSV。不得绑定其他用户上传的附件。
4. 管理员：审批活动、审批发起人、创建/抓取数据源、查看抓取日志及失败原因、维护词表和知识节点。

### 关键安全与异常检查

- 错误验证码不能登录或重置密码；验证码一次性使用且重复发送受限流。
- 未登录访问草稿列表返回 403；其他用户不能读草稿、编辑活动、读取报名表或下载草稿附件。
- 附件上传限制类型和大小；发布前仅所有者/管理员可下载，发布后可公开下载。
- 推荐接口不返回报名表中的学号、学院、联系邮箱；推荐包含理由，并排除已报名活动。
- 分别检查 401、403、404、422、500 的提示及重试行为。

## 五、数据库迁移

新数据库：

```powershell
$env:AUTO_CREATE_TABLES='false'
$env:DATABASE_URL='sqlite:///migration-test.db'
.\.venv\Scripts\flask.exe --app wsgi db upgrade
.\.venv\Scripts\flask.exe --app wsgi db current
```

已有数据库必须先备份。旧库无 Alembic 版本记录时，从后端仓库根目录运行：

```powershell
python scripts\upgrade_legacy_schema.py --legacy-bootstrap
```

该命令会补齐缺失表/兼容字段后标记为当前迁移版本，不删除业务数据。

## 六、Docker/PostgreSQL（安装 Docker 后）

```powershell
cd D:\ZhangChenrui\Projects\Campus-Activity-Information-Platform\backend
docker compose up -d --build
docker compose exec api flask --app wsgi db current
```

再执行第三部分的角色验收。结束时使用 `docker compose down`，不要使用 `-v`，否则会删除 PostgreSQL 和附件持久卷。
