# 前端本地测试指南

最后更新：2026-07-14。

## 启动

```bash
npm install
node mock/index.js
npm run dev
```

Mock 服务监听 `5000`，Vite 监听 `3000`。请求通过 `/api` 代理转发；不要再使用已删除的 `mock-server.js`，也不要使用旧路由 `/login` 或 `/activities/:id` 查看详情。

## 自动化检查

```bash
npm run verify
```

它会运行类型检查与生产构建、Vitest 单元测试、Playwright 端到端测试。端到端测试会自动启动 Mock 与 Vite 服务。

## 手动验收

1. 访客访问首页，执行搜索、分类筛选和详情查看。
2. 使用 `zhangsan / 123456` 登录，分别访问“我的报名”和“我的创建”；在“我的创建”的已通过、在审核、未通过三个独立页面间切换。上传一份允许类型的附件（≤10 MB），创建活动并提交审核；确认提交后进入“在审核”页面。
3. 清除登录状态后使用 `admin / admin123456` 登录，在审核队列批准或驳回活动。
4. 在活动详情完成报名，确认活动自动出现在“我报名的活动”和个人日历；取消报名后确认两处同步删除。继续验证收藏、ICS 导出、订阅关联节点，并在个人中心确认通知已读状态。
5. 在 `/activities` 和 `/search` 切换排序，确认 URL 与结果同步；在被驳回活动中确认显示审核反馈。
6. 在 375px、768px、1024px 和桌面宽度检查无横向溢出，并用 Tab 键访问主要按钮、日历活动和通知。

## 后端联调

1. 停止 Mock，启动后端服务。
2. 复制 `.env.example` 为 `.env.local`，按部署方式设置 `VITE_API_BASE_URL`；同源反向代理通常保持 `/api`。
3. 保持前端 `Activity` 与 `/activities` 格式，由网关/API 适配层转换后端资源。
4. 按 [后端联调清单](frontend-plan/BACKEND_INTEGRATION_CHECKLIST.md) 验证认证、时区、分页、错误码、下载和角色边界。

后端总文档未冻结的报名、独立收藏、资料更新和密码重置能力，不应在联调中被误判为后端已实现；其前端 Mock 流程只用于页面验收。
